# Third-Party Software and Dependencies

---

## English

[➡ Switch to Deutsch](#deutsch)

### Purpose

This document lists all third-party software, libraries, tools, and assets
used within the **Flowline** project.

Its purpose is to ensure:
- License compliance
- Transparency for users and contributors
- Traceability for audits and industrial environments

### Scope

This document covers:
- Runtime dependencies
- Development and build-time dependencies
- UI frameworks and tooling
- Documentation tools, if redistributed

It does **not** cover:
- Operating systems
- Compilers or SDKs not redistributed with Flowline

### Rules for Introducing Dependencies

Before adding a new third-party dependency, the following rules apply:

- The license **must be compatible** with Apache License 2.0
- The dependency **must not impose copyleft obligations**
- The dependency **must not restrict commercial or industrial use**
- The dependency **must be actively maintained** or technically justified

### Documentation Requirements

Each dependency must be documented with:

- Name
- Version
- License
- Usage context (runtime, build, test, docs)
- Source URL

### Current Dependencies

> At the moment, Flowline does not include any third-party dependencies.

This section will be updated as dependencies are introduced.

### Responsibility

Each contributor is responsible for:
- Declaring newly introduced dependencies
- Verifying license compatibility
- Updating this document accordingly

## Deutsch

[➡ Switch to English](#english)

### Zweck

Dieses Dokument listet alle Drittanbieter-Software, Bibliotheken, Tools und Assets auf,
die im Projekt **Flowline** verwendet werden.

Ziel ist:
- Lizenzkonformität
- Transparenz für Nutzer und Mitwirkende
- Nachvollziehbarkeit für Audits und industrielle Umgebungen

### Geltungsbereich

Dieses Dokument umfasst:
- Laufzeitabhängigkeiten
- Entwicklungs- und Build-Abhängigkeiten
- UI-Frameworks und Tooling
- Dokumentationswerkzeuge, sofern weiterverteilt

Nicht umfasst sind:
- Betriebssysteme
- Compiler oder SDKs, die nicht mit Flowline ausgeliefert werden

### Regeln für neue Abhängigkeiten

Vor der Einführung einer neuen Drittanbieter-Abhängigkeit gelten folgende Regeln:

- Die Lizenz **muss kompatibel** zur Apache License 2.0 sein
- Die Abhängigkeit **darf keine Copyleft-Pflichten** erzwingen
- Die Abhängigkeit **darf kommerzielle oder industrielle Nutzung nicht einschränken**
- Die Abhängigkeit **muss aktiv gepflegt** oder technisch begründet sein

### Dokumentationspflicht

Jede Abhängigkeit muss dokumentiert werden mit:

- Name
- Version
- Lizenz
- Nutzungskontext (Runtime, Build, Test, Doku)
- Quell-URL

### Aktuelle Abhängigkeiten

> Aktuell enthält Flowline keine Drittanbieter-Abhängigkeiten.

Dieser Abschnitt wird erweitert, sobald Abhängigkeiten hinzukommen.

### Verantwortung

Jeder Mitwirkende ist verantwortlich für:
- das Deklarieren neu eingeführter Abhängigkeiten
- die Prüfung der Lizenzkompatibilität
- die Pflege dieses Dokuments
